# RoomDB

- 애플리케이션 전체에서 하나의 DB만 생성할 수 있도록 보장하기 위해 싱글톤 패턴 사용
- 애플리케이션 전체에서 공유할 수 있는 객체

## synchronized 사용 이유

`synchronized` 키워드는 멀티스레딩 환경에서 동시에 실행될 수 있는 코드 블록을 한 번에 하나의 스레드만 접근할 수 있도록 보호하는 역할을 합니다.

`getInstance` 메서드에서 `synchronized(AppDatabase::class.java)` 블록을 볼 수 있습니다.

이는 `getInstance` 메서드가 동시에 여러 스레드에서 호출될 수 있다는 가능성을 고려하여 작성되었습니다.

만약 `synchronized`를 사용하지 않았다면 `getInstance` 메서드가 여러 스레드에서 동시에 호출될 수 있고, `AppDatabase` 객체를 동시에 생성할 수 있습니다.

그러나 `AppDatabase`는 애플리케이션에 한 개만 존재할 수 있어햐 합니다.



더 구체적으로 설명하자면, `synchronized` 키워드는 블록 내부의 모든 코드가 락(lock) 기법으로 보호됩니다.

즉, 한 스레드가 블록 내부의 코드를 실행하는 동안 다른 스레드는 해당 블록에 진입하지 못하고 대기(wait)하게 됩니다.

이를 통해 동기화 문제를 해결합니다.
